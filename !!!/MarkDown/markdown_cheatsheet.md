# Markdown — шпаргалка

> Markdown — это язык разметки для оформления текста. Файлы имеют расширение `.md`.
> Используется в README, документации, докстрингах (через инструменты), GitHub, GitLab и т.д.

---

## 1. Заголовки (и «размер шрифта»)

В Markdown **нет прямого управления размером шрифта** — размер определяется уровнем заголовка. Чем больше `#`, тем мельче.

```markdown
# H1 — самый крупный
## H2
### H3
#### H4
##### H5
###### H6 — самый мелкий
```

Альтернативный синтаксис для H1 и H2:

```markdown
Заголовок H1
============

Заголовок H2
------------
```

> Если нужен **произвольный размер шрифта или цвет** — используйте HTML (см. раздел 14).

## 2. Выделение текста (emphasis)

| Что нужно | Синтаксис | Результат |
|-----------|-----------|-----------|
| Курсив | `*текст*` или `_текст_` | *текст* |
| Жирный | `**текст**` или `__текст__` | **текст** |
| Жирный курсив | `***текст***` | ***текст*** |
| Зачёркнутый | `~~текст~~` | ~~текст~~ |
| Подчёркнутый | только HTML: `<u>текст</u>` | <u>текст</u> |

```markdown
Обычный, *курсив*, **жирный**, ***жирный курсив***, ~~зачёркнутый~~.
```

## 3. Код

### Inline-код (внутри строки)

```markdown
Используй функцию `pytest.raises()` для проверки исключений.
```
Используй функцию `pytest.raises()` для проверки исключений.

### Блок кода с подсветкой синтаксиса

Укажите язык после трёх обратных кавычек (` ``` `):

````markdown
```python
import pytest


def test_division_by_zero():
    with pytest.raises(ZeroDivisionError):
        _ = 1 / 0
```
````

Результат:

```python
import pytest


def test_division_by_zero():
    with pytest.raises(ZeroDivisionError):
        _ = 1 / 0
```

Часто используемые языки: `python`, `bash`, `json`, `yaml`, `sql`, `html`, `gherkin`.

## 4. Списки

### Маркированные

```markdown
- Pytest
- Playwright
  - Вложенный пункт (2 пробела или таб)
    - Ещё глубже
* Можно и звёздочкой
+ И плюсом
```

### Нумерованные

```markdown
1. Установить зависимости
2. Написать тест
3. Запустить `pytest`
   1. Вложенная нумерация
```

> Номера можно ставить любые — Markdown пересчитает сам. Но лучше писать по порядку.

### Чек-листы (task lists, работает на GitHub)

```markdown
- [x] Написать BaseAPI
- [x] Добавить типизацию
- [ ] Покрыть тестами
- [ ] Настроить CI
```

- [x] Написать BaseAPI
- [ ] Покрыть тестами

## 6. Изображения

Синтаксис как у ссылки, но с `!` в начале:

```markdown
![Альтернативный текст](path/to/image.png)
![Логотип](https://example.com/logo.png "Подсказка")
```

Размер изображения в чистом Markdown задать нельзя — только через HTML:

```markdown
<img src="logo.png" width="200" alt="Логотип">
```

## 8. Таблицы

```markdown
| Метод | Описание | Возвращает |
|-------|----------|-----------:|
| `get()`  | GET-запрос    | `Response` |
| `post()` | POST-запрос   | `Response` |
```

Выравнивание задаётся двоеточиями в строке-разделителе:

```markdown
| По левому | По центру | По правому |
|:----------|:---------:|-----------:|
| left      | center    | right      |
```

## 9. Горизонтальная линия

Три или более символа на отдельной строке:


## 10. Переносы строк

- **Один** перенос строки в коде даёт **пробел** (строки склеиваются).
- Чтобы сделать **жёсткий перенос**, поставьте **два пробела** в конце строки или `<br>`.
- Чтобы начать **новый абзац** — оставьте **пустую строку**.

```markdown
Первая строка.  ← два пробела в конце
Вторая строка.

Новый абзац.
```

---

## 11. Экранирование символов

Чтобы вывести спецсимвол как обычный текст, поставьте перед ним `\`:

```markdown
\*это не курсив\*
\# это не заголовок
```

Экранируются: `\` `` ` `` `*` `_` `{}` `[]` `()` `#` `+` `-` `.` `!` `|`

## 14. HTML внутри Markdown (размер шрифта, цвет)

Markdown допускает HTML — так решается вопрос размера и цвета шрифта:

```markdown
<span style="font-size: 24px; color: red;">Крупный красный текст</span>

<sub>нижний индекс</sub> и <sup>верхний индекс</sup>

<kbd>Ctrl</kbd> + <kbd>C</kbd>
```

> ⚠️ На GitHub `style="..."` вырезается из соображений безопасности — цвет/размер через инлайн-стили там не сработает. Работает в большинстве рендереров (VS Code, локальные превью), но не на GitHub.

### Якоря и оглавление

Заголовки автоматически становятся якорями (в нижнем регистре, пробелы → дефисы):

```markdown
## Запуск тестов  →  ссылка: [перейти](#запуск-тестов)
```

### Документирование примера запуска

````markdown
## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
pytest -v --tb=short
```
````

## Шпаргалка-минимум

| Действие | Синтаксис |
|----------|-----------|
| Заголовок | `# H1`, `## H2`, … |
| Жирный | `**текст**` |
| Курсив | `*текст*` |
| Зачёркнутый | `~~текст~~` |
| Inline-код | `` `код` `` |
| Блок кода | ` ```python … ``` ` |
| Список | `- пункт` |
| Нумерованный | `1. пункт` |
| Чек-лист | `- [ ]` / `- [x]` |
| Ссылка | `[текст](url)` |
| Картинка | `![alt](url)` |
| Цитата | `> текст` |
| Линия | `---` |
| Таблица | `\| a \| b \|` |
